# SKILL: Travel Shared Table (Full Spec)

## 目的
建立旅遊「航班與住宿共用表」平台，包含登入權限、場次代碼管理、航班住宿列表、表單新增/編輯、登入背景設定，並部署於公開網域。

## 功能規格

### 1. 登入與權限
- 角色：管理者 / 一般登錄者
- 管理者可設定一般登錄者密碼
- 一般登錄者只需密碼登入（無帳號）
- Session 有效期 30 分鐘
- 關閉瀏覽器即失效

### 2. 場次代碼管理（管理者專屬）
- 新增、修改、刪除
- 修改需按「修改儲存」才生效
- 刪除需判斷是否已有資料，有則跳確認視窗
- 刪除後不可再出現在列表與新增頁面

### 3. 列表功能
- 使用單一表格顯示所有資料
- 欄位包含：
  操作、場次代碼、姓名、去程出發日期、去程出發地點、去程抵達日期、去程抵達地點、住宿飯店、住宿狀態、回程出發日期、回程出發地點、回程抵達日期、回程抵達地點
- 時間顯示為「日期 + 時間」
- 搜尋條件：場次代碼、出發地點、住宿狀態
- 顯示筆數
- 排序：去程出發日期時間 → 建立時間
- 刪除僅管理者可操作

### 4. 新增 / 編輯表單
- 去程 / 回程區塊標示
- 出發/抵達日期時間 + 地點
- 姓名旁註記「社群內名字非真名」
- 住宿飯店、住宿狀態
- 日期時間欄位為單一欄位元件（可選日期與時間）
- 儲存後更新資料

### 5. 登入畫面
- 背景圖片由管理者上傳
- 登入卡片透明度 50%
- 背景區塊滿版到視窗高度
- 左上標題可由管理者設定並同步到瀏覽器分頁名稱

### 6. 技術與部署
- 前端：Next.js
- DB：Supabase Postgres
- 部署：Vercel
- auth：JWT + cookie
- DB 欄位建議使用 is_destroyed 進行軟刪除
- 加入 /api/health 健康檢查（會打 DB）
- 以排程每週三/六呼叫 /api/health，避免 Supabase 閒置暫停

---

## 交付目標
- 可用於多次旅遊活動
- 支援多人同時使用
- 管理者可控所有設定與資料
